141// Create a RateLimiter object, which can be shared among RocksDB instances to
142// control write rate of flush and compaction.
143// @rate_bytes_per_sec: this is the only parameter you want to set most of the
144// time. It controls the total write rate of compaction and flush in bytes per
145// second. Currently, RocksDB does not enforce rate limit for anything other
146// than flush and compaction, e.g. write to WAL.
147// @refill_period_us: This controls how often tokens are refilled. For example,
148// when `rate_bytes_per_sec` is set to 10MB/s and
149// `refill_period_us` is set to 100ms, then 1MB is refilled
150// every 100ms internally. Larger values can lead to sporadic
151// delays while smaller values introduce more CPU overhead.
152// @fairness: RateLimiter accepts high-pri requests and low-pri requests.
153// A low-pri request is usually blocked in favor of hi-pri request. Currently,
154// RocksDB assigns low-pri to request from compaction and high-pri to request
155// from flush. Low-pri requests can get blocked if flush requests come in
156// continuously. This fairness parameter grants low-pri requests permission by
157// 1/fairness chance even though high-pri requests exist to avoid starvation.
158// You should be good by leaving it at default 10.
159// @mode: Mode indicates which types of operations count against the limit.
160// @auto_tuned: Enables dynamic adjustment of rate limit within the range
161// `[rate_bytes_per_sec / 20, rate_bytes_per_sec]`, according to
162// the recent demand for background I/O.
163// @single_burst_bytes: The maximum number of bytes that can be granted in a
164// single call to `Request()`. Zero is a special value
165// meaning the number of bytes per refill.
166RateLimiter* NewGenericRateLimiter(
167 int64_t rate_bytes_per_sec, int64_t refill_period_us = 100 * 1000,
168 int32_t fairness = 10,
169 RateLimiter::Mode mode = RateLimiter::Mode::kWritesOnly,
170 bool auto_tuned = false, int64_t single_burst_bytes = 0);
