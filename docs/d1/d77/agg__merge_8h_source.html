15// The feature is still in development so the encoding format is subject
16// to change.
17//
18// Aggregation Merge Operator is a merge operator that allows users to
19// aggregate merge operands of different keys with different registered
20// aggregation functions. The aggregation can also change for the same
21// key if the functions store the data in the same format.
22// The target application highly overlaps with merge operator in general
23// but we try to provide a better interface so that users are more likely
24// to use pre-implemented plug-in functions and connect with existing
25// third-party aggregation functions (such as those from SQL engines).
26// In this case, the need for users to write customized C++ plug-in code
27// is reduced.
28// If the idea proves to useful, we might consider to move it to be
29// a core functionality of RocksDB, and reduce the support of merge
30// operators.
31//
32// Users can implement aggregation functions by implementing abstract
33// class Aggregator, and register it using AddAggregator().
34// The merge operator can be retrieved from GetAggMergeOperator() and
35// it is a singleton.
36//
37// Users can push values to be updated with a merge operand encoded with
38// registered function name and payload using EncodeAggFuncAndPayload(),
39// and the merge operator will invoke the aggregation function.
40// An example:
41//
42// // Assume class ExampleSumAggregator is implemented to do simple sum.
43// AddAggregator("sum", std::make_unique<ExampleSumAggregator>());
44// std::shared_ptr<MergeOperator> mp_guard = CreateAggMergeOperator();
45// options.merge_operator = mp_guard.get();
46// ...... // Creating DB
47//
48//
49// std::string encoded_value;
50// s = EncodeAggFuncAndPayload(kUnamedFuncName, "200", encoded_value);
51// assert(s.ok());
52// db->Put(WriteOptions(), "foo", encoded_value);
53// s = EncodeAggFuncAndPayload("sum", "200", encoded_value);
54// assert(s.ok());
55// db->Merge(WriteOptions(), "foo", encoded_value);
56// s = EncodeAggFuncAndPayload("sum", "200", encoded_value);
57// assert(s.ok());
58// db->Merge(WriteOptions(), "foo", encoded_value);
59//
60// std::string value;
61// Status s = db->Get(ReadOptions, "foo", &value);
62// assert(s.ok());
63// Slice func, aggregated_value;
64// assert(ExtractAggFuncAndValue(value, func, aggregated_value));
65// assert(func == "sum");
66// assert(aggregated_value == "600");
67//
68//
69// DB::Put() can also be used to add a payloadin the same way as Merge().
70//
71// kUnamedFuncName can be used as a placeholder function name. This will
72// be aggregated with merge operands inserted later based on function
73// name given there.
74//
75// If the aggregation function is not registered or there is an error
76// returned by aggregation function, the result will be encoded with a fake
77// aggregation function kErrorFuncName, with each merge operands to be encoded
78// into a list that can be extracted using ExtractList();
79//
80// If users add a merge operand using a different aggregation function from
81// the previous one, the merge operands for the previous one is aggregated
82// and the payload part of the result is treated as the first payload of
83// the items for the new aggregation function. For example, users can
84// Merge("plus, 1"), merge("plus 2"), merge("minus 3") and the aggregation
85// result would be "minus 0".
86//
87
88// A class used to aggregate data per key/value. The plug-in function is
89// implemented and registered using AddAggregator(). And then use it
90// with merge operator created using CreateAggMergeOperator().
91class Aggregator {
92 public:
93 virtual ~Aggregator() {}
94 // The input list is in reverse insertion order, with values[0] to be
95 // the one inserted last and values.back() to be the one inserted first.
96 // The oldest one might be from Get().
97 // Return whether aggregation succeeded. False for aggregation error.
98 virtual bool Aggregate(const std::vector<Slice>& values,
99 std::string& result) const = 0;
100
101 // True if a partial aggregation should be invoked. Some aggregators
102 // might opt to skip partial aggregation if possible.
103 virtual bool DoPartialAggregate() const { return true; }
104};

106// The function adds aggregation plugin by function name. It is used
107// by all the aggregation operator created using CreateAggMergeOperator().
108// It's currently not thread safe to run concurrently with the aggregation
109// merge operator. It is recommended that all the aggregation function
110// is added before calling CreateAggMergeOperator().
111Status AddAggregator(const std::string& function_name,
112 std::unique_ptr<Aggregator>&& agg);
113
114// Get the singleton instance of merge operator for aggregation.
115// Always the same one is returned with a shared_ptr is hold as a
116// static variable by the function.
117// This is done so because options.merge_operator is shared_ptr.
118std::shared_ptr<MergeOperator> GetAggMergeOperator();

120// Encode aggregation function and payload that can be consumed by aggregation
121// merge operator.
122Status EncodeAggFuncAndPayload(const Slice& function_name, const Slice& payload,
123 std::string& output);
124// Helper function to extract aggregation function name and payload.
125// Return false if it fails to decode.
126bool ExtractAggFuncAndValue(const Slice& op, Slice& func, Slice& value);
127
128// Extract encoded list. This can be used to extract error merge operands when
129// the returned function name is kErrorFuncName.
130bool ExtractList(const Slice& encoded_list, std::vector<Slice>& decoded_list);
