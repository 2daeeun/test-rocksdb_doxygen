16/*
17 * This is a simplified version of MergingIterator and is specifically used for
18 * compaction. It merges the input `children` iterators into a sorted stream of
19 * keys. Range tombstone start keys are also emitted to prevent oversize
20 * compactions. For example, consider an L1 file with content [a, b), y, z,
21 * where [a, b) is a range tombstone and y and z are point keys. This could
22 * cause an oversize compaction as it can overlap with a wide range of key space
23 * in L2.
24 *
25 * CompactionMergingIterator emits range tombstone start keys from each LSM
26 * level's range tombstone iterator, and for each range tombstone
27 * [start,end)@seqno, the key will be start@seqno with op_type
28 * kTypeRangeDeletion unless truncated at file boundary (see detail in
29 * TruncatedRangeDelIterator::start_key()).
30 *
31 * Caller should use CompactionMergingIterator::IsDeleteRangeSentinelKey() to
32 * check if the current key is a range tombstone key.
33 * TODO(cbi): IsDeleteRangeSentinelKey() is used for two kinds of keys at
34 * different layers: file boundary and range tombstone keys. Separate them into
35 * two APIs for clarity.
36 */
37class CompactionMergingIterator;
