108 // Memtable Garbage Collection algorithm: a MemPurge takes the list
109 // of immutable memtables and filters out (or "purge") the outdated bytes
110 // out of it. The output (the filtered bytes, or "useful payload") is
111 // then transfered into a new memtable. If this memtable is filled, then
112 // the mempurge is aborted and rerouted to a regular flush process. Else,
113 // depending on the heuristics, placed onto the immutable memtable list.
114 // The addition to the imm list will not trigger a flush operation. The
115 // flush of the imm list will instead be triggered once the mutable memtable
116 // is added to the imm list.
117 // This process is typically intended for workloads with heavy overwrites
118 // when we want to avoid SSD writes (and reads) as much as possible.
119 // "MemPurge" is an experimental feature still at a very early stage
120 // of development. At the moment it is only compatible with the Get, Put,
121 // Delete operations as well as Iterators and CompactionFilters.
122 // For this early version, "MemPurge" is called by setting the
123 // options.experimental_mempurge_threshold value as >0.0. When this is
124 // the case, ALL automatic flush operations (kWRiteBufferManagerFull) will
125 // first go through the MemPurge process. Therefore, we strongly
126 // recommend all users not to set this flag as true given that the MemPurge
127 // process has not matured yet.
128 Status MemPurge();
129 bool MemPurgeDecider(double threshold);
130 // The rate limiter priority (io_priority) is determined dynamically here.
131 Env::IOPriority GetRateLimiterPriority();

134 // Require db_mutex held.
135 // Called only when UDT feature is enabled and
136 // `persist_user_defined_timestamps` flag is false. Because we will refrain
137 // from flushing as long as there are still UDTs in a memtable that hasn't
138 // expired w.r.t `full_history_ts_low`. However, flush is continued if there
139 // is risk of entering write stall mode. In that case, we need
140 // to track the effective cutoff timestamp below which all the udts are
141 // removed because of flush, and use it to increase `full_history_ts_low` if
142 // the effective cutoff timestamp is newer. See
