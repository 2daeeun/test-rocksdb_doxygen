59// Reader class for BlockBasedTable format.
60// For the format of BlockBasedTable refer to
61// https://github.com/facebook/rocksdb/wiki/Rocksdb-BlockBasedTable-Format.
62// This is the default table type. Data is chucked into fixed size blocks and
63// each block in-turn stores entries. When storing data, we can compress and/or
64// encode data efficiently within a block, which often results in a much smaller
65// data size compared with the raw data size. As for the record retrieval, we'll
66// first locate the block where target record may reside, then read the block to
67// memory, and finally search that record within the block. Of course, to avoid
68// frequent reads of the same block, we introduced the block cache to keep the
69// loaded blocks in the memory.

758 // through `xsputn()` e.g. `std::endl` or an unsigned long long is written by

760 // int_type __CLR_OR_THIS_CALL sputc(_Elem _Ch) { // put a character

765 // not captured by xsputn(), becomes an overflow here.

769 if (s.ok()) {

771 }
772 }
773 return EOF;
774 }

777 Status s = file_->Append(Slice(p, n));
