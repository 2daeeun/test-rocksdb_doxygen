76 // Keep the current key-value as-is.
77 kKeep,
78
79 // Remove the current key-value. Note that the semantics of removal are
80 // dependent on the value type. If the current key-value is a plain
81 // key-value or a wide-column entity, it is converted to a tombstone
82 // (Delete), resulting in the deletion of any earlier versions of the key.
83 // If it is a merge operand, it is simply dropped. Note: if you are using
84 // a TransactionDB, it is not recommended to filter out merge operands.
85 // If a Merge operation is filtered out, TransactionDB may not realize there
86 // is a write conflict and may allow a Transaction that should have failed
87 // to Commit. Instead, it is better to implement any Merge filtering inside
88 // the MergeOperator.
89 kRemove,
90
91 // Change the value of the current key-value. If the current key-value is a
92 // plain key-value or a merge operand, its value is updated but its value
93 // type remains the same. If the current key-value is a wide-column entity,
94 // it is converted to a plain key-value with the new value specified.
95 kChangeValue,
96
97 // Remove all key-values with key in [key, *skip_until). This range of keys
98 // will be skipped in a way that potentially avoids some IO operations
99 // compared to removing the keys one by one. Note that removal in this case
100 // means dropping the key-value regardless of value type; in other words, in
101 // contrast with kRemove, plain values and entities are not converted to
102 // tombstones.
103 //
104 // *skip_until <= key is treated the same as Decision::kKeep (since the
105 // range [key, *skip_until) is empty).
106 //
107 // Caveats:
108 // * The keys are skipped even if there are snapshots containing them,
109 // i.e. values removed by kRemoveAndSkipUntil can disappear from a
110 // snapshot - beware if you're using TransactionDB or DB::GetSnapshot().
111 // * If value for a key was overwritten or merged into (multiple Put()s
112 // or Merge()s), and `CompactionFilter` skips this key with
113 // kRemoveAndSkipUntil, it's possible that it will remove only
114 // the new value, exposing the old value that was supposed to be
115 // overwritten.
116 // * Doesn't work with PlainTableFactory in prefix mode.
117 // * If you use kRemoveAndSkipUntil for table files created by compaction,
118 // consider also reducing compaction_readahead_size option.
